# Tabloom 视觉与文案规范

## 1. 已确认的视觉方向

采用“精致深色工作台”的卡片层次与信息密度，同时默认跟随系统主题；系统为浅色时显示浅色背景，系统为深色时显示深色背景。用户可以手动覆盖为 Light 或 Dark。

布局规则：

- 桌面端完整页面，而非手机宽度弹窗。
- 三个业务区块保持上、中、下纵向结构。
- 每个区块横向铺满主内容区。
- 区块内部的网站卡片可根据宽度显示三列或两列。
- 主要断点：1180px 以下卡片两列；页面最低支持宽度为 900px。

## 2. 颜色变量

所有颜色集中在 `dashboard.css` 顶部的 CSS Variables 中。修改主题应优先调整变量，不要在组件中散落新的硬编码颜色。

浅色主题：

- 页面背景：`--bg: #f5f7f8`
- 卡片背景：`--panel: #ffffff`
- 主文字：`--text: #192026`
- 品牌强调：`--accent: #779500`
- Open Tabs 强调：`--violet: #7055b8`
- 风险操作：`--danger: #c84e5c`

深色主题：

- 页面背景：`--bg: #0b1117`
- 卡片背景：`--panel: #17212b`
- 主文字：`--text: #f2f5f6`
- 品牌强调：`--accent: #b7dc31`
- Open Tabs 强调：`--violet: #b090ff`
- 风险操作：`--danger: #ff8490`

## 3. 组件规则

- 区块圆角：14px。
- 网站卡片圆角：11px。
- 控件圆角：7–10px。
- 页面标题和摘要必须单行截断，防止卡片高度失控。
- favicon 优先使用网页资源；加载失败时显示网站名称首字母。
- 普通图标默认低强调，卡片 hover 后提高可见度。
- Undo 只使用常见撤销图标，辅助信息通过 tooltip 和 `aria-label` 提供。
- 批量危险操作不应使用大面积红色；仅在 hover、确认弹窗和最终确认按钮中加强风险提示。

## 4. 英文 UI 词汇表

以下词汇已结合产品语境确认，修改时保持一致：

| 中文含义 | 使用文案 |
| --- | --- |
| 待处理网页 | `For Later` |
| 已打开的网页 | `Open Tabs` |
| 收藏的网页 | `Saved` |
| 全部处理完成 | `Complete all` |
| 关闭全部 | `Close all` |
| 移除全部收藏 | `Remove all` |
| 处理完成 | `Mark as done` |
| 稍后处理 | `Keep for later` |
| 收藏网页 | `Save page` |
| 撤销 | 图标 + tooltip `Undo` |

产品自身的按钮、说明、空状态和确认弹窗统一使用英文。网页标题、摘要、站点名属于网页内容，应保持其原始语言。

## 5. 可访问性

- 图标按钮必须提供 `aria-label`。
- 颜色不能成为唯一的状态提示。
- 所有主要操作应支持键盘聚焦。
- 遵循 `prefers-reduced-motion`，减少动画用户不播放过渡动画。
- Light/Dark 主题都应保持文字与背景的清晰对比。

## 6. 动效与声音

- 单项完成：约 260ms，轻微缩小、强调色确认后退出。
- 单项关闭：约 220ms，向右短距离滑出。
- 批量操作：网站卡片按 42ms 间隔错峰退出，总等待时间设有上限。
- 撤销：卡片从上方轻微回弹进入。
- 收藏/稍后处理：图标短暂弹性放大并出现扩散圆环。
- 关键操作在点击位置显示约 650ms 的光圈和粒子反馈；关闭为风险色、完成和收藏为品牌强调色、撤销为紫色。
- 光标效果只在操作触发时出现，不使用持续跟随拖尾，避免长时间干扰。
- 声音控制必须保留 Off；默认 Subtle，Playful 仅提高音量和存在感。
- 不给搜索输入、刷新、页面切换等高频动作增加声音。
