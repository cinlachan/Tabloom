# Tabloom 产品与交互规格

## 1. 产品定位

Tabloom 是面向“经常同时打开大量网页”的用户设计的标签页工作台。它不只是展示当前标签页，还帮助用户区分三类状态：现在正在使用、稍后需要处理、值得长期保留。

产品名：**Tabloom**（Tab + Loom）  
品牌语：**Keep tabs clear. Save what matters.**

## 2. 核心原则

1. 信息优先：用户应一眼看清网页标题、摘要、网站和当前状态。
2. 状态独立：`For Later` 和 `Saved` 不依赖标签页是否仍然打开。
3. 操作就近：单项操作放在网页行右侧，批量操作放在对应区块右上方。
4. 风险可控：全量关闭或移除需要确认；执行后提供区块顶部撤销。
5. 保留网页原貌：网页标题和摘要保持网页自身语言，不进行翻译。
6. 本地优先：当前版本不依赖账号和后端，数据只保存在浏览器本地。

## 3. 页面信息架构

三个区块必须按纵向从上到下排列，不能改为左、中、右三栏：

1. `For Later`
2. `Open Tabs`
3. `Saved`

每个区块包含：

- 折叠/展开按钮
- 区块名称与数量
- 一句状态说明
- 操作反馈与图标式 Undo
- 区块级批量操作
- 按网站聚合的卡片

## 4. For Later

用途：保存当前没时间处理、之后需要阅读、回复或完成的网页。

- 从 Open Tabs 点击时钟图标加入。
- 即使原标签页关闭，条目仍然保留。
- 点击条目：如果网页仍打开，切换到原标签页；否则重新打开 URL。
- 勾选表示处理完成，并从本区移除。
- 单项操作：打开、标记完成、收藏。
- 网站级操作：`Complete site`。
- 全部操作：`Complete all`，执行前确认。
- 撤销：恢复刚移除的条目。

## 5. Open Tabs

用途：管理所有浏览器窗口中当前打开的网页。

- 数量与浏览器窗口数显示在区块说明中。
- 按网站聚合，相同网站的多个标签页放在同一卡片。
- 卡片显示网站 favicon/logo、网站名和标签页数量。
- 网页行显示原始标题与网页自身摘要。
- 单项操作：切换、加入 For Later、加入 Saved、关闭。
- 网站级操作：`Close site`。
- 全部操作：`Close all`，执行前确认；Tabloom 页面自身不关闭。
- 撤销：重新打开刚关闭网页的 URL，不恢复浏览历史。

Open Tabs 是符合英语产品语境的名词短语，表示“当前处于打开状态的标签页”，不要改为 `Opened Tabs`。

## 6. Saved

用途：长期保留值得收藏的网页。

- 从 Open Tabs 或 For Later 点击书签图标加入。
- 原标签页关闭后仍然保留。
- 点击条目：切换到已打开标签页或重新打开 URL。
- 单项操作：打开、移除。
- 网站级操作：`Remove site`。
- 全部操作：`Remove all`，执行前确认。
- 移除收藏不应关闭正在打开的标签页。

## 7. 搜索

- 搜索范围：网页标题、摘要、网站名称和 URL。
- 搜索同时过滤三个区块。
- 快捷键：macOS 使用 `⌘K`，Windows/Linux 使用 `Ctrl+K`。
- 无匹配内容时显示统一空状态，并提供 `Clear search`。

## 8. 动效与声音反馈

- 声音模式：`Off`、`Subtle`、`Playful`，默认 `Subtle`。
- 完成、收藏、关闭、批量清理、撤销和折叠拥有不同的短反馈。
- 批量处理只播放一次组合声音，禁止为每个条目重复播放。
- 声音由浏览器本地实时生成，不加载外部音频、不联网。
- 系统开启“减少动态效果”时，跳过退出等待并关闭主要过渡动画。
- 搜索、刷新和普通网页切换等高频动作默认不播放声音。

## 9. 后续可扩展方向

- 为 For Later 添加截止日期、优先级和备注。
- 支持拖拽排序和自定义分组。
- 支持导入、导出和跨设备同步。
- 支持标签页自动休眠、重复标签页检测。
- 支持站点黑名单或仅按需读取网页摘要，以降低权限范围。
- Chrome Web Store 正式发布及隐私政策页面。
